시맨틱 웹 관련 요소

개요

☞ "나는 검색된다. 고로 존재한다"는 말이 있을 정도로 웹사이트는 검색엔진으로의 노출이 매우 중요함

☞ SEO(검색엔진 최적화: Search Engine Optimization)같은 마케팅 도구를 사용하여 검색엔진이 본인의 웹사이트를 검색하기 알맞은 구조로 웹사이트를 조정해야 할 필요 존재

☞ 기본적으로 검색엔진이 웹사이트 정보를 어떻게 수집하는지 잘 아는 것이 중요!

☞ 검색엔진은 로봇(Robot)이라는 프로그램을 이용해 매일 전세계의 웹사이트 정보를 수집함

☞ 검색 로봇이 수집하는 정보는 결국 웹사이트의 HTML 코드를 기반으로 작성됨

☞ 즉, 검색 엔진은 HTML코드 만으로 그 의미를 인지하여야 하는데, 이때 시맨틱Semantic 요소 여부에 따라 의미가 다르게 전달될 수 있음

☞ 시맨틱 요소로 구성되어 있는 웹페이지는 검색엔진에 보다 의미론적으로 문서 정보를 전달할 수 있고 검색엔진 또한 시맨틱 요소를 이용하여 보다 효과적인 크롤링과 인덱싱이 가능하게 됨

☞ 시맨틱 태그란 브라우저, 검색엔진, 개발자 모두에게 콘텐츠의 의미를 명확히 설명하는 역할 담당

☞ 시맨틱 태그에 의해 컴퓨터가 HTML 요소의 의미를 보다 명확히 해석하고 데이터로 활용할 수 있는 시맨틱 웹을 실현할 수 있음

☞ 시맨틱 웹이란 웹에 존재하는 수많은 웹페이지들에 메타데이터(Metadata)를 부여하여 기존의 잡다한 데이터 집합이었던 웹페이지를 '의미'와 '관련성'을 가지는 새로운 정보를 만들어 낼 수 있도록 만든 차세대 지능형 웹

시맨틱태그

header : 웹사이트 로고, 검색폼을 나타내는 영역

nav : 웹사이트를 구성하는 메뉴gnb를 나타내는 영역

main : 웹사이트의 중요 컨텐츠를 표시하는 영역

section : 중요 컨텐츠를 구성하는 하위 컨텐츠를 그룹화할 때 사용

article : 중요 컨텐츠들을 구성하는 하위 컨텐츠가 보여지는 영역

aside : 웹사이트 중요 컨텐츠와 관련 없는 내용을 표시하는 영역

footer : 웹사이트 저작권이나 관리자 정보 등을 나타내는 영역
